# Internationalization

Default product locale:

```txt
zh-TW
```

Supported MVP locales:

```txt
zh-TW
en-US
```

Angular build configuration uses Angular's supported `zh-Hant` locale data for the Traditional Chinese source build while preserving `zh-TW` as the product locale and HTML language.

All visible Angular template text must use `i18n` markers. TypeScript user-facing strings must use `$localize`.

Required files:

```txt
apps/web/src/locale/messages.xlf
apps/web/src/locale/messages.zh-TW.xlf
apps/web/src/locale/messages.en-US.xlf
```

Required extraction command:

```bash
pnpm --filter web extract-i18n
```

Angular CLI equivalent from `apps/web`:

```bash
ng extract-i18n --output-path src/locale
```

## Formatting

Use Angular pipes or tested custom formatters for money, dates, and numbers.

Required display examples:

```txt
NT$1,280
2026/06/13
1,280
```

Do not manually format date, number, or currency strings unless the formatter is tested.
